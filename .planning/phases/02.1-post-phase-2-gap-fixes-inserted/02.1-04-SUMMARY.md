# Phase 02.1 Plan 04: Admin Problem Tab + Icon Previews + ContextProblem API Fetch Summary

**One-liner:** Problem tab with 12-field stat card editor wired to site_content.php, plus 20px ICON_REGISTRY SVG previews in Hero tab and ContextProblem live data fetch with hardcoded fallback.

## What Was Built

### Task 1: Admin site-content page — Icon previews + Problem tab

**File:** `frontend/src/app/admin/site-content/page.tsx`

Changes:
- Added `import { ICON_REGISTRY } from "@/components/blocks/floating-icons-hero-demo"`
- Added `ProblemContent` interface with 12 fields (`stat1_value`, `stat1_detail`, `stat1_title`, `stat1_body` × 3 cards)
- Extended `TabId` type to `"hero" | "nav" | "stats" | "footer" | "problem"`
- Added `DEFAULT_PROBLEM` constant matching backend seed values
- Added `"Problem"` entry to TABS array
- Added `problemContent` and `problemStatus` state declarations
- Extended `fetchAll` useEffect Promise.all to fetch `section=problem`; handles response with `setProblemContent`
- Extended `saveSection()` union type to accept `ProblemContent`
- Updated Hero tab icon slot JSX — added `flex` row with 20px SVG preview (IIFE lookup `ICON_REGISTRY[slot.icon]`) beside the existing `<select>` dropdown
- Added full Problem tab JSX block: 3 stat card rows each with 4 inputs (Value, Detail, Title, Body) using template literal key indexing, plus Save Problem button with saved/error feedback states

### Task 2: ContextProblem — API fetch on mount

**File:** `frontend/src/components/blocks/context-problem.tsx`

Changes:
- Added `{ useState, useEffect }` to React import
- Added `safeFetch` import from `@/lib/utils`
- Added `ProblemStats` interface (12 fields)
- Added `FALLBACK_PROBLEM_STATS` constant with identical hardcoded values
- Added `ps` state initialized to `FALLBACK_PROBLEM_STATS`
- Added `useEffect` that calls `safeFetch("/api/site_content.php?section=problem")` on mount; merges API data over fallback on success; keeps fallback on error
- Replaced all 3 StatCard prop triplets (`stat`, `detail`, `title`, `body`) with `ps.statN_*` dynamic values
- StatCard component, BeforePanel, AfterPanel remain unchanged
