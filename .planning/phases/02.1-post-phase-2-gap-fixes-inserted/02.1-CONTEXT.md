<decisions>
## Implementation Decisions

### Fix 1: Icon Slot Visual Previews (Admin)
- **File:** `frontend/src/app/admin/site-content/page.tsx`
- **File:** `frontend/src/components/blocks/floating-icons-hero-demo.tsx` (needs to export `ICON_REGISTRY`)
- **What:** The Hero tab shows 6 icon slot dropdowns with text-only names ("openai", "n8n" etc.). Render the actual SVG icon (20px) next to each dropdown option so the admin can see what icon they're choosing.
- **How:** Export `ICON_REGISTRY` from `floating-icons-hero-demo.tsx`. In admin page Hero tab, import it and render `<Icon size={20} />` beside/inside each dropdown.

### Fix 2: Hero Eyebrow Field
- **Files:** `frontend/src/components/blocks/floating-icons-hero-section.tsx`, `frontend/src/components/blocks/floating-icons-hero-demo.tsx`, `frontend/src/app/admin/site-content/page.tsx`
- **What:** `FloatingIconsHeroProps` has no `eyebrow` prop. Add it and render above the heading.
- **How:**
  - Add `eyebrow?: string` to `FloatingIconsHeroProps` interface in `floating-icons-hero-section.tsx`
  - Render eyebrow above title (small uppercase label, muted color, similar to other section eyebrows)
  - Add `eyebrow` field to `HeroContent` interface and `DEFAULT_HERO` const in admin page
  - Add "Eyebrow Label" text input in the Hero tab of admin site-content page
  - Wire eyebrow field to the site_content API save/load (section: "hero", key: "eyebrow")

### Fix 3: Homepage Section Eyebrow Text
- **Files:** `frontend/src/components/blocks/converged-platform.tsx`, `frontend/src/components/blocks/agency-features.tsx`, `frontend/src/components/blocks/workflow-demo.tsx`, `frontend/src/components/blocks/testimonials.tsx`, `frontend/src/components/blocks/context-problem.tsx`, `frontend/src/components/blocks/agency-stats.tsx`
- **What:** All sections should have clear, descriptive eyebrow badge text. Review current values; `WorkflowDemo` likely has none.
- **Locked eyebrow text values:**
  - `ContextProblem`: "The Automation Gap"
  - `AgencyStats`: "Proven Results"
  - `Testimonials`: "Client Success Stories"
  - `ConvergedPlatform`: "Our Platform"
  - `AgencyFeatures`: "Why Choose Us"
  - `WorkflowDemo`: "How It Works" (add eyebrow — none currently exists)

### Fix 4: Problem Section Stats in Admin
- **Files:** `backend/api/site_content.php`, `frontend/src/app/admin/site-content/page.tsx`, `frontend/src/components/blocks/context-problem.tsx`
- **What:** The 3 stat cards in ContextProblem (14 hrs, 6+ tools, 40%) are hardcoded. Make them editable from admin.
- **Backend:** Add `problem` section support to `site_content.php`. Keys needed: `stat1_value`, `stat1_detail`, `stat1_title`, `stat1_body`, `stat2_value`, `stat2_detail`, `stat2_title`, `stat2_body`, `stat3_value`, `stat3_detail`, `stat3_title`, `stat3_body`. Reuse existing `site_content` table.
- **Admin:** Add "Problem" tab to admin site-content page. Show 3 stat card rows, each with 4 inputs (stat value, stat detail, title, body text). Wire to `site_content.php` `update_section` POST action for section "problem".
- **Frontend:** `context-problem.tsx` fetches `site_content.php?section=problem` on mount via `safeFetch`. Falls back to current hardcoded values if API fails or returns error.

### Fix 5: Footer Newsletter Column Layout
- **File:** `frontend/src/components/ui/footer-section.tsx`
- **Current issue:** Brand column is `lg:w-[38%]` with logo + short tagline (`max-w-[28ch]`) + social icons + newsletter form. The narrow tagline vs wide newsletter form looks unbalanced.
- **Locked fix:** Restructure to a **4-column equal grid** at lg breakpoint:
  - Column 1: Brand (logo + tagline + social icons only — no newsletter form here)
  - Column 2: Navigation links group 1
  - Column 3: Navigation links group 2
  - Column 4: Newsletter signup form (full column width)
  - All 4 columns equal width (`lg:grid-cols-4`)
  - On mobile: stacks vertically, newsletter at bottom

- Whether to use `<Badge>` component or `<span>` for eyebrow in hero (match whichever pattern the other sections use)
