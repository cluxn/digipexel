## Spacing Scale

Declared values (all multiples of 4, extracted from existing patterns in `globals.css`, `connect-cta.tsx`, `services/[slug]/page.tsx`):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, pill `gap-1`, `gap-2` tight inline spacing |
| sm | 8px | Input field padding (label to input), `gap-3` for pill row |
| md | 16px | Default form field gap, `gap-4` card grid |
| lg | 24px | Section inner padding, `gap-6` between heading and body |
| xl | 32px | `gap-8` between major form groups, card grids |
| 2xl | 48px | `py-12` section bottom padding |

**Source:** Derived from `connect-cta.tsx` (py-24, px-10, space-y-5, gap-5), `services/[slug]/page.tsx` (gap-12, gap-20, mb-6, mb-10, mt-8, pt-32, pb-12), `globals.css` (--radius-sm: 4px, --radius-md: 8px).

## Section-by-Section Visual Contract

### Public Service Page Sections

Every section follows: eyebrow Badge → two-part heading (line1 / line2 accent) → max 2-sentence body copy. This is the SVC-01/SVC-08 design contract.

#### Hero Section

- Layout: 2-column grid at lg (`grid-cols-[1.1fr_0.9fr]`), single column below
- Min-height: `min-h-[90vh]`
- Background: `bg-base` with `radial-gradient` ambient light (top-left)
- Eyebrow: `Badge variant="outline"` with `.section-eyebrow` class — `service.badge` text
- Heading: `h1` with `.hero-title` — `heroLine1` on line 1, `heroLine2` in `<span className="hero-title-accent">` on line 2
- Body: `text-lg md:text-xl font-medium text-secondary opacity-70 leading-relaxed` — `service.heroCopy` (max 2 sentences)
- Primary CTA button: `Button asChild variant="brand"` → `Link href="/contact-us"` — label = `service.ctaPrimary`
- No secondary CTA button rendered (D-20 locked)
- Pills row: `flex flex-wrap gap-3`, each pill = `rounded-xl bg-highlight border border-border-subtle text-[10px] font-bold uppercase tracking-widest` with `CheckCircle2 w-3 h-3 text-brand` icon
- Right panel: snapshot card (`snapshotTitle`, 4 `snapshotRows`) + 2 stat chips (`statLabel1/statValue1`, `statLabel2/statValue2`)

#### Comparison / Gap Section

- Layout: 2-column comparison at md breakpoint, single column below
- Stays hardcoded (D-13) — not connected to API
- Left column: "Traditional" approach (red/warning styling)
- Right column: "Our approach" (brand/green styling)
- No eyebrow badge needed — `gapHeading` serves as section title

#### Platform / Features Section

- Eyebrow: hardcoded `"The Platform"` badge
- Heading: `platformTitle` / `platformTitleAccent` — split heading pattern
- 3 feature cards: `rounded-2xl border border-border-subtle bg-surface p-8` — each has hardcoded icon (Lucide), `title`, `description`
- Feature card bottom: `Button asChild variant="brand" size="sm"` → `Link href="/contact-us"` with text "Get Started →"
- Icon: hardcoded by card index (D-14) — `w-8 h-8 text-brand`

#### Roadmap Section

- Eyebrow: hardcoded `"The Delivery Process"` badge
- Heading: `roadmapTitle` + `roadmapTitleAccent` from service_content API
- Body: `roadmapCopy` (max 2 sentences)
- 6 steps: `grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-6` — each step card: step number badge (`text-brand font-black`), title, `desc` (2-line max guideline)
- Step number: `text-4xl font-black text-brand/20` (large muted background number)

#### Market Impact / Outcomes Section

- Eyebrow: hardcoded `"Market Impact"` badge
- Heading: `outcomesTitle` + `outcomesTitleAccent` from service_content API
- Body: `outcomesCopy` (max 2 sentences)
- 2 outcome cards: quote + company + sector chip + metric value/label
- 4 stat chips in a row: `value` (large, `text-brand`) + `label` (small, `text-secondary`)

#### Testimonials Section

- Eyebrow: hardcoded `"Real Intelligence"` badge
- 2–3 testimonial cards from service_content API: `quote` + `role` + `company`
- Card style: `rounded-2xl bg-surface border border-border-subtle p-8` — quote in italic body, attribution in label size

#### CTA Section (Connect Component)

- Uses `<Connect variant="light" />` extended with optional `badge?`, `title?`, `copy?` props
- When API data provides `ctaBadge`/`ctaTitle`/`ctaCopy` from service_content, pass as props to override defaults
- When API unavailable, Connect renders its hardcoded defaults — acceptable fallback
- Button inside Connect: "Start Building Now" → `/contact-us`

### Admin Services Page

#### Service Selector

- Position: top of page content, above tabs
- Element: `<select>` styled consistently with the admin page pattern — `h-10 px-3 rounded-md border border-border-subtle bg-surface text-sm font-medium`
- Label: `"Select Service"` (`field-label` class: `text-[10px] font-black uppercase tracking-widest text-slate-500`)
- On change: re-fetch all 6 sections for new slug; reset all content states; no unsaved-changes warning (consistent with site-content page behavior per RESEARCH.md Pitfall 3)

#### Tab Navigation

- 6 tabs: Hero | Features | Roadmap | Market Impact | CTA | Testimonials
- Active tab: `bg-brand text-white rounded-md` or `border-b-2 border-brand` — match `admin/site-content` active tab style
- Inactive tab: `text-secondary hover:text-primary`
- Tab bar: `flex gap-1 border-b border-border-subtle mb-8`

#### Form Fields (all tabs)

- Input: `w-full rounded-md border border-border-subtle bg-base px-3 py-2 text-sm focus:outline-none focus:ring-2 focus:ring-brand/20`
- Textarea: same as input + `resize-none`, rows as appropriate (2 rows for desc fields, 4 rows for longer copy)
- Label: `.field-label` utility — `text-[10px] font-black uppercase tracking-widest text-slate-500 mb-1`
- Field spacing: `space-y-4` within a tab, grouped fields in `grid grid-cols-1 md:grid-cols-2 gap-4`
- Section grouping: `border border-border-subtle rounded-xl p-6 space-y-4 mb-6` for grouped row sets (e.g., each roadmap step, each testimonial)

#### Save Button + Status

- Button: `Button variant="brand" size="sm"` — label "Save Hero", "Save Features", etc. (tab-specific)
- Status states (3-state per tab): idle (button visible) → `"Saving…"` (button disabled, spinner) → `"Saved"` (green checkmark text) or `"Error saving"` (red text)
- Auto-reset to idle after 2.5s — same `autoReset()` pattern from `admin/site-content/page.tsx`

## Interaction States

### Public Service Page

| State | Trigger | Visual Treatment |
|-------|---------|-----------------|
| Page load | Immediate | Render static `SERVICES[slug]` — no loading blank. API data silently replaces state when resolved. |
| API success | `safeFetch` resolves with `status: "success"` | State updates in place — no visible transition for user |
| API failure | Network error or non-2xx | Static fallback remains; no error shown to visitor |
| Button hover | Mouse over `.btn-brand` | `scale(1.02)` + glow shadow (defined in `.btn-brand:hover`) |
| Button active | Mouse down | `active:scale-[0.95]` |
| Pill chip | Static | No interaction — display only |

### Admin Services Page

| State | Trigger | Visual Treatment |
|-------|---------|-----------------|
| Initial load | Component mount | Fetch sections for default slug (`"ai-seo"`); show `"Synchronizing Data…"` spinner |
| Service selector change | `onChange` on `<select>` | Re-fetch all sections; show loading; reset all form states |
| Tab switch | Click tab | Immediate — no fetch needed; content already in state |
| Field change | Input/textarea `onChange` | Controlled state update — no debounce |
| Save click | Button `onClick` | Button → `"Saving…"` disabled; POST to `service_content.php`; resolve to `"Saved"` or `"Error saving"` |
| Auto-reset | 2.5s after save/error | Status returns to idle; button re-enables |
