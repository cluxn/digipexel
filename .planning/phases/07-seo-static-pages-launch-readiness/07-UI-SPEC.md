## Spacing Scale

Declared values (multiples of 4 only — 8-point scale in use across all prior phases):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline label gaps |
| sm | 8px | Compact element spacing, input group gaps |
| md | 16px | Default element spacing, form field gaps |
| lg | 24px | Section internal padding, card padding |
| xl | 32px | Layout column gaps, block-level vertical rhythm |
| 2xl | 48px | Section vertical padding (`py-12`) |
| 3xl | 64px | Major section breaks (`py-16`, `py-24` on CTA blocks) |

Exceptions:
- Touch targets (CalendlyButton trigger): minimum 44px height — use `h-12` (48px) as closest step-up, matching existing brand button height
- Admin SEO page notice banner: `p-4` (16px) inner padding, `mb-6` (24px) bottom margin — keeps it visually distinct without introducing new spacing values
- Legal page body text paragraphs: `mb-6` (24px) between paragraphs — standard prose rhythm

Source: spacing confirmed from `globals.css`, `button.tsx` (`h-12`, `h-14`, `px-8`), `connect-cta.tsx` (`py-24`, `px-6`, `p-12 md:p-20`)

### STATIC-02: Custom 404 Page

File to create: `app/not-found.tsx`

Layout: Inherits root layout (Navbar + Footer from `app/layout.tsx`) — do NOT import them manually.

Visual contract:
- Primary focal point: `.hero-title` heading (largest element; draws eye first)
- Badge: `.section-eyebrow` class — text: "404 — Signal Lost"
- Heading: `.hero-title` class — "Lost in the <span class='hero-title-accent'>automation void</span>?" (Claude's call per CONTEXT.md D-05)
- Subheading: `.section-subtitle` class — "This page doesn't exist in our system. Let's get you back on track."

- CTA row (flex wrap, justify-center, gap-4):

  - Secondary: `Button variant="outline"` → `href="/services"` — label: "View Our Services"
- Section padding: `py-32` top (clears navbar), `py-16` bottom
- Max width: `max-w-2xl mx-auto px-6`

Constraints: Server Component (no `"use client"`) — avoids hydration issues. No Navbar/Footer import.

### SEO-05: CalendlyButton Component

File to create: `components/ui/calendly-button.tsx`

Visual contract:
- Renders `react-calendly` `PopupWidget` as the trigger
- `textColor`: `"#ffffff"` (white text)
- `color`: `"#7C3AED"` (brand violet — NOT `#2563eb` from library default)

## Interaction Contract

### CalendlyButton — Trigger States

| State | Visual |
|-------|--------|
| `calendly_url` not set | Renders as standard `<Link>` with `Button variant="brand"` — identical to current behavior |
| `calendly_url` set | Renders Calendly `PopupWidget` trigger button styled with `color="#7C3AED"` |
| Widget loading | Calendly handles its own loading state within the modal |
| Widget open | Full-height overlay modal from `react-calendly` — no custom styling needed |

### Admin SEO Page — Form States

| State | Visual |
|-------|--------|
| Initial load (page selected) | Fetches saved meta; fields populate with saved values or remain empty |
| Loading saved meta | Inputs disabled, `opacity-50`, `Loader2` spinner inline |
| Dirty (unsaved changes) | No indicator needed — save is explicit via button |
| Save in progress | Button shows "Saving…", disabled, `opacity-50` |
| Save success | Inline text below button: "Meta saved. Rebuild to apply changes." in `text-semantic-success` |
| Save error | Inline text below button: "Save failed. Check your connection and try again." in `text-destructive` |

### 404 Page — No Interactive States

Static page. No loading states, no form. CTA buttons are standard link navigations.
